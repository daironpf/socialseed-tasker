# SocialSeed Tasker - Injected Infrastructure

This directory contains the Tasker management layer for external projects that need to integrate with SocialSeed Tasker.

## Structure

```
tasker/
├── skills/              # AI Agent skill functions for Tasker API interaction
├── configs/             # Environment configuration (.env)
└── docker-compose.yml   # Neo4j database for local development
```

## Quick Start

### 1. Start the Database

```bash
# Local mode (Docker)
docker compose up -d

# Check health
docker compose ps
```

### 2. Configure Environment

```bash
cp configs/.env.example configs/.env
# Edit configs/.env with your settings
```

### 3. Initialize Tasker

```bash
# Scaffold Tasker files into your project
tasker init .
```

### 4. Use the Skills

Import and use the AI agent skill functions directly:

```python
import sys
sys.path.insert(0, "tasker/skills")

from task_skill import (
    create_issue,
    list_issues,
    update_issue,
    close_issue,
    add_dependency,
    get_dependency_chain,
    get_workable_issues,
    analyze_impact,
    create_component,
    get_project_summary
)

# Create an issue
result = create_issue(
    title="Implement feature X",
    component_id="<uuid>",
    priority="HIGH"
)

# List issues
issues = list_issues(status="OPEN")

# Get workable issues
workable = get_workable_issues()

# Analyze impact
impact = analyze_impact(issue_id="<uuid>")
```

## Switching to Neo4j Aura

Edit `configs/.env` and configure for cloud:

```bash
# Neo4j Aura (cloud)
TASKER_NEO4J_URI=bolt+s://your-id.databases.neo4j.io:7687
TASKER_NEO4J_USER=neo4j
TASKER_NEO4J_PASSWORD=your_aura_password

# Or Local (Docker)
TASKER_NEO4J_URI=bolt://localhost:7687
TASKER_NEO4J_USER=neo4j
TASKER_NEO4J_PASSWORD=neoSocial
```

## Configuration Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `TASKER_API_URL` | Yes | Base URL for Tasker API (default: http://localhost:8000) |
| `TASKER_API_KEY` | No | API key for authentication |
| `TASKER_NEO4J_URI` | Yes | Neo4j connection URI |
| `TASKER_NEO4J_USER` | Yes | Neo4j username |
| `TASKER_NEO4J_PASSWORD` | Yes | Neo4j password |

## AI Agent Workflow Example

```python
import sys
sys.path.insert(0, "tasker/skills")

from task_skill import (
    create_issue,
    get_workable_issues,
    update_issue,
    close_issue,
    get_dependency_chain
)

def implement_feature(feature_name, component_id):
    """AI agent implements a feature following Tasker workflow."""
    
    # 1. Find workable issues
    workable = get_workable_issues(component=component_id)
    
    # 2. Select and start working
    issue = workable[0]
    update_issue(issue["id"], {
        "status": "IN_PROGRESS",
        "agent_working": True
    })
    
    # 3. Do the work...
    
    # 4. Close and summarize
    close_issue(issue["id"])
    update_issue(issue["id"], {
        "agent_working": False,
        "description": f"## Solution\nImplemented {feature_name}"
    })

# Example: Work on issues for backend component
backend = list_components(project="my-app")[0]
implement_feature("Cache layer", backend["id"])
```

## For AI Agents

This scaffolded layer allows AI agents to interact with Tasker without direct imports. They use HTTP calls to the REST API via the functions in `task_skill.py`. This decouples the agent code from Tasker's internal structure.

**Key benefits:**
- No direct imports of Tasker core required
- Agent code is portable to other projects
- Easy to swap between local and cloud Neo4j
- Complete CRUD and analysis capability
